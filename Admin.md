# Admin Panel - Full Specification

## Overview

Админ-панель предоставляет полный контроль над каталогом книг, пользователями, арендами, покупками, штрафами, контентом и аналитикой системы.

---

## 1. Управление книгами

### 1.1 Список книг
- Таблица всех книг с фильтрацией и сортировкой
- Колонки: ID, Название, ISBN, Формат (Physical/Digital), Наличие, Цены, Статус
- Фильтры: по формату, доступности, категории, автору
- Поиск по названию, ISBN, автору

### 1.2 Создание книги
- Заполнение метаданных:
  - Название, описание, ISBN
  - Год издания, обложка (URL)
  - has_physical, has_digital
  - physical_inventory, digital_licenses (-1 = безлимит)
  - price_purchase, price_rental, deposit_amount
  - is_available_for_rent, is_available_for_purchase
- Загрузка PDF файла (в S3/MinIO)
- Привязка авторов (из существующих или создание новых)
- Привязка категорий (из существующих или создание новых)

### 1.3 Редактирование книги
- Полное обновление всех полей
- Частичное обновление (отдельные поля)
- Обновление цен (отдельный экран)
- Изменение количества копий (инвентаризация)
- Переключение доступности
- Замена PDF файла

### 1.4 Удаление книги
- Soft delete (isActive = false) — скрытие из каталога
- Hard delete — полное удаление (только если нет активных аренд/покупок)
- Предупреждение при наличии связанных данных

### 1.5 Аналитика по книге
- Количество аренд (`total_rentals_count`)
- Количество покупок (`total_purchases_count`)
- Средний рейтинг
- Список отзывов
- Список пользователей, арендовавших книгу
- История статусов

## 2. Управление пользователями

### 2.1 Список пользователей
- Таблица пользователей с фильтрацией
- Колонки: ID, Имя, Email, Роли, Статус, Задолженность, Дата регистрации
- Фильтры: по роли, статусу, задолженности
- Поиск по имени, email

### 2.2 Профиль пользователя
- Личные данные
- История аренд
- История покупок
- История штрафов
- Баланс задолженности
- Активные сессии чтения
- История действий

### 2.3 Управление ролями
- Назначение ролей (ROLE_USER, ROLE_ADMIN)
- Снятие ролей
- Валидация ролей из БД

### 2.4 Статус пользователя
- Активация / деактивация
- Бан / разбан
- Просмотр причины бана

### 2.5 Ручное управление штрафами
- Начисление штрафа вручную (с указанием причины)
- Снятие/списание штрафа (waive)
- История изменений штрафов

### 2.6 GDPR — Анонимизация
- Запрос на удаление аккаунта
- Анонимизация данных:
  - email → `deleted_{uuid}@deleted.local`
  - username → `deleted_user_{uuid}`
  - name, phone → NULL
  - password → random hash
- Записи в rentals, fines, purchases сохраняются
- user_debt сохраняется до погашения

## 3. Управление арендами и возвратами

### 3.1 Список активных аренд
- Таблица аренд со статусами
- Колонки: ID, Пользователь, Книга, Тип, Статус, Дата начала, Дата окончания, Дней осталось
- Фильтры: по статусу, типу, пользователю
- Поиск по книге, пользователю

### 3.2 Сканирование штрих-кода
- Ввод receipt_code вручную или сканером
- Автоматический поиск аренды/покупки
- Отображение информации:
  - Пользователь, книга
  - Тип операции (аренда/покупка)
  - Статус
  - Сумма, даты
- Кнопки действий:
  - Подтвердить выдачу (аренда)
  - Подтвердить возврат (аренда)
  - Подтвердить оплату (покупка)

### 3.3 Подтверждение возврата
- Сканирование штрих-кода → нахождение аренды
- Расчёт штрафа (если есть просрочка)
- Отображение суммы штрафа
- Обработка депозита:
  - Если штраф = 0: Release всего холдирования
  - Если штраф < депозит: Capture штрафа, Release остатка
  - Если штраф >= депозит: Capture депозита, разница → user_debt
- Обновление статуса → RETURNED
- Increment `physical_inventory` (для Physical)
- Increment `digital_licenses` (для Digital, если не -1)

### 3.4 Просроченные аренды
- Список аренд со статусом OVERDUE
- Колонки: Пользователь, Книга, Дней просрочки, Сумма штрафа
- Кнопка «Пометить как Lost» (при достижении cap)

### 3.5 Ручное закрытие аренд
- Закрытие аренды админом (с указанием причины)
- Обработка штрафов и депозита
- Обновление инвентаря

---

## 4. Управление покупками

### 4.1 Список покупок
- Таблица всех покупок
- Колонки: ID, Пользователь, Книга, Тип, Способ оплаты, Статус, Цена, Дата
- Фильтры: по статусу, типу, способу оплаты
- Поиск по книге, пользователю

### 4.2 Подтверждение cash-покупок
- Сканирование штрих-кода → нахождение покупки
- Подтверждение оплаты
- Обновление статуса → COMPLETED
- Decrement `physical_inventory` (для Physical)
- Генерация PdfCopy с анти-пиратским кодом (для Digital)

### 4.3 История транзакций
- Все транзакции через Payment Service
- Фильтрация по статусу, дате, пользователю

---

## 5. Управление штрафами

### 5.1 Список штрафов
- Таблица всех штрафов
- Колонки: ID, Пользователь, Книга (аренда), Сумма, Статус, Способ оплаты, Дата
- Фильтры: по статусу, пользователю
- Поиск по пользователю, книге

### 5.2 Ручное начисление штрафа
- Выбор пользователя
- Выбор аренды
- Ввод суммы и причины
- Создание записи в `fines`
- Начисление в `user_debt`

### 5.3 Снятие штрафа (Waive)
- Выбор штрафа
- Указание причины снятия
- Обновление статуса → WAIVED
- Обновление `user_debt.total_debt`

### 5.4 Подтверждение cash-оплаты штрафов
- Поиск штрафа по пользователю
- Подтверждение оплаты
- Обновление статуса → PAID
- Обновление `user_debt.total_debt`
- Если `total_debt = 0` → снятие блокировки

### 5.5 Отчёт по задолженностям
- Общая сумма задолженностей
- Количество пользователей с задолженностью
- Топ должников
- История погашений

---

## 6. Управление контентом

### 6.1 CRUD авторов
- Создание, редактирование, удаление авторов
- Поля: full_name, bio, photo_url
- Привязка к книгам через `book_authors`

### 6.2 CRUD категорий
- Создание, редактирование, удаление категорий
- Поля: name, parent_id (иерархия), description
- Привязка к книгам через `book_categories`

### 6.3 Модерация отзывов
- Список отзывов со статусом PENDING
- Действия: APPROVED / REJECTED
- Причина rejection (опционально)
- Список всех отзывов с фильтрацией по статусу

### 6.4 Управление промокодами
- Создание промокодов:
  - code, discount_type (PERCENT/FIXED), discount_value
  - applicable_to (RENTAL/PURCHASE/BOTH)
  - max_uses, valid_from, valid_until
- Редактирование промокодов
- Деактивация промокодов
- Аналитика использования (promo_code_usages)

---

## 7. Аналитика и отчёты

### 7.1 Графики популярности книг
- Топ арендованных книг
- Топ покупаемых книг
- Динамика по периодам (день/неделя/месяц/год)

### 7.2 Самые невозвращаемые книги
- Рейтинг книг по количеству просрочек
- Рейтинг книг по количеству потерь (Lost)

### 7.3 Выручка за период
- Общая выручка (аренды + покупки)
- Разбивка по типам (Physical/Digital)
- Разбивка по способам оплаты (CARD/CASH)
- Динамика по периодам

### 7.4 Отчёт по просрочкам и штрафам
- Количество просрочек за период
- Общая сумма начисленных штрафов
- Общая сумма оплаченных штрафов
- Общая сумма списанных штрафов (waived)

### 7.5 Отчёт по задолженностям пользователей
- Общая сумма задолженностей
- Количество пользователей с задолженностью
- Топ должников
- Динамика задолженностей

### 7.6 Статистика по форматам
- Соотношение Physical vs Digital аренд
- Соотношение Physical vs Digital покупок
- Использование цифровых лицензий

### 7.7 Аналитика по промокодам
- Какой промокод сколько раз использован
- Какая сумма скидок по каждому промокоду
- Эффективность промокодов (конверсия)

---

## 8. Инвентаризация

### 8.1 Списание книг
- Причины: утерянная, ветхая, повреждённая
- Decrement `physical_inventory`
- Запись в лог инвентаризации

### 8.2 Добавление новых поступлений
- Increment `physical_inventory`
- Increment `digital_licenses`
- Указание источника поступления

### 8.3 Сверка наличия
- Экспорт текущего инвентаря
- Ввод фактического наличия
- Автоматический расчёт расхождений
- Подтверждение расхождений (списание/добавление)

---

## 9. События от Admin Panel

| Событие | Описание |
|---------|----------|
| `BookCreated` | Новая книга создана |
| `BookUpdated` | Книга обновлена |
| `BookDeleted` | Книга удалена |
| `ReturnConfirmed` | Возврат подтверждён |
| `FineCharged` | Штраф начислен |
| `FineWaived` | Штраф снят |
| `FinePaidConfirmed` | Оплата штрафа подтверждена |
| `PurchaseConfirmed` | Покупка подтверждена (cash) |
| `UserBanned` | Пользователь забанен |
| `UserUnbanned` | Пользователь разбанен |
| `UserAnonymizationRequested` | Запрос на анонимизацию |
| `ReviewApproved` | Отзыв утверждён |
| `ReviewRejected` | Отзыв отклонён |
| `PromoCodeCreated` | Промокод создан |
| `InventoryAdjusted` | Инвентарь скорректирован |
